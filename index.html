What T2T changed in the human reference.

A genome-wide delta, computed by an agent-run sandbox harness.

Abstract. The T2T-CHM13 assembly closes many gaps in GRCh38, especially centromeric, pericentromeric, acrocentric, and other repeat-rich sequence. I compared GRCh38 and T2T-CHM13 with the same CpG-island kernel used in the reference-broadcast demo: warm the reference once, snapshot the sandbox, fork one shard per chromosome, and reduce. T2T adds 179.6 Mb of usable sequence and 40,492 additional candidate CpG islands. The added sequence is not compositionally uniform: some chromosomes gain mostly island-poor heterochromatin, while acrocentric chromosomes gain CpG-rich sequence that the classical island rule calls heavily. The result is not a new biological discovery; it is a compact, reproducible measurement of what the more complete reference changes, and a useful stress test for agent-orchestrated genomics.

The surprising part is the sign pattern. I expected the newly resolved sequence to be mostly CpG-island poor. That is true for large heterochromatic additions such as chrY and chr9. But it is not true genome-wide. The acrocentric chromosomes, especially chr13, chr15, chr21, and chr22, gain many candidate islands. That is exactly where a simple Gardiner-Garden and Frommer rule is expected to be vulnerable: GC/CpG-rich rDNA and satellite contexts can satisfy a sequence-composition rule without being ordinary promoter islands.

Interpretation: T2T adds both dark matter and false-positive bait. The correct story is not "T2T is island-rich" or "T2T is island-poor." It is that a complete reference exposes two different classes of previously unresolved sequence, and a simple CpG-island caller separates them imperfectly but visibly.

Method: the harness is the agent

This follow-up uses the same systems primitive as the first paper, but points it at T2T-CHM13. The control plane is not a scheduler. It is a Claude Code agent running ./crabbox.sh t2t: download and split the complete hs1 FASTA, warm one sandbox, snapshot it, restore-and-fork one shard per chromosome on islo.dev, collect JSON, reduce, and clean up.

04

What this does and does not claim

This is a reference comparison. It compares public assemblies, not individuals. It says nothing about any person, trait, diagnosis, or methylation state.

The islands are candidates. The 1987 CpG-island rule is useful because it is transparent, but it over-calls in some GC-rich repeat contexts. That is not hidden here; it is part of the result.

The new signal is methodological. The follow-up is interesting because it uses a reproducible agent harness to expose a known hard edge in genome annotation: completing the reference changes what simple sequence-composition rules see.

The result is inspectable. Every number is in data/compare.json. The site renders from that file at load time.
